## 一、深度解析 Minecraft 皮肤结构

Minecraft 皮肤本质上是一张包含 RGBA 四个通道的纹理贴图，常见规格为 64×64 像素（早期版本为 64×32 像素）。
贴图被划分为多个区域，这些区域按照特定方式展开，对应模型的某个部位的一个面。

![Minecraft 皮肤 UV 贴图|900](/articles/images/skingen_uv_map.jpg)

每个部位又分为内层和外层两种类型：内层通常用来表示基本身体，外层通常用来表现头发的层次感、外套或盔甲（也有例外，当立体结构比较复杂时，内层也可能用于装饰层的补充）。
外层皮肤在尺寸上略大于内层，配合透明像素的支持，它能够刻画出具有 3D 层次感的发型、服饰轮廓以及丰富的细节装饰。

- **头部**：内/外层均为 8×8×8。外层像素宽度为内层的 9/8 倍，使其相对于内层在各方向上突出 0.5 像素。
- **躯干**：内/外层均为 8×12×4。外层像素宽度为内层的 8.5/8 倍，高度为内层的 12.5/12 倍，使其在各方向上突出 0.25 像素。
- **手臂与腿部**：内/外层均为 4×12×4。外层像素宽度为内层的 4.5/4 倍（如果是细手臂的正面和背面，则是 3.5/3 倍），高度为内层的 12.5/12 倍，使其在各方向上突出 0.25 像素。

![Minecraft 皮肤内外层尺寸示意|800](/articles/images/skingen_layers.jpg)

需要注意的是，所有内层皮肤均由等宽的正方形像素块组成，但躯干和四肢的外层皮肤为了保持比例协调，并非完全由正方形像素块组成。而且，头部、躯干、四肢外层皮肤的缩放比例也存在细微差别。
这种复杂的空间映射逻辑，以及不同视角下内外层皮肤的结构约束，使通用 AI 难以稳定生成可直接使用的皮肤贴图。

## 二、通用大模型的边界

在进入微调训练之前，我们先尝试使用若干通用多模态图像模型（例如 OpenAI 的 GPT Image2 和 Google 的 Nano Banana）来直接完成皮肤生成任务。

这一步的目标并不是比较模型的审美能力，而是验证通用模型能否同时满足三个工程条件：

- 输出符合 Minecraft 64×64 皮肤 UV 布局的贴图；
- 将参考图中的身份特征、服饰和配色合理映射到头部、躯干、四肢和外层结构；
- 生成可被后处理流程稳定解析，并最终导入游戏的 PNG 皮肤文件。

### 为什么目前通用多模态图像模型难以直接生成可用的 Minecraft 皮肤？
基于这些测试，我们归纳出四类主要问题：

- **第一，复杂指令遵循能力不足。**

Minecraft 皮肤的 UV 贴图结构约束极其严格。64×64 图像中的每一个像素都有精确的空间含义。

头部、身体、手臂、腿部的贴图在 UV 贴图上的位置和方向是严格规定的。一个像素的偏差，在 3D 模型上就会表现为贴图错位。

这类模型难以仅使用提示词（或者参考图）约束生成图像，使其稳定遵循像素级精确的结构。

- **第二，对复杂三维结构的理解能力不足。**

我们不仅需要生成准确的 UV 贴图，还要使纹理映射到三维空间时尽量保留原图像的特征，并考虑不同观察角度下的遮挡关系、组件之间的遮挡关系、各关节和组件的距离等细节。例如围巾这类装饰通常会跨多个组件，如果不能正确理解空间关系，最终效果会出现缺失或错位。

- **第三，透明通道的缺失。**

Minecraft 外层皮肤可使用透明通道来表达哪些像素的显示和不显示。

这决定了帽子的形状、外套的轮廓和头发的层次。

通用图像模型通常不会稳定输出可直接使用的透明通道。如果使用特定颜色作为背景，也会在皮肤颜色与背景颜色相近时干扰前景和背景的区分。

## 三、微调训练的技术选型

仅依靠提示词工程难以胜任皮肤生成的任务，我们转而尝试微调训练。

在探索过程中，我们注意到了 Cory Spencer 在 Monadical 网站上公开的 Minecraft 皮肤生成系列文章。该方案设计了一种复合输出图像布局：一部分是 UV 贴图，另一部分是 3D 渲染图。这种设计有助于引导模型学习二维 UV 贴图与三维立体模型之间的对应关系，并在一定程度上缓解像素级结构生成和复杂三维结构理解的问题。在我们的测试中，单独训练 UV 贴图生成或单独训练立体模型渲染图，效果都不够稳定；将二者结合后，模型在这个任务上的表现有明显提升。

![Monadical 方案示意|800](/articles/images/monadical.jpg)

不过，该方案也存在一些局限性。“文生皮肤（Text-to-Skin）”是基于“文生图（Text-to-Image）”基础模型微调而来的，这类模型很难直接迁移到“图生皮肤（Image-to-Skin）”任务中，因为我们很难用纯文本完整描述一张图片的所有细节。但“图生皮肤”功能对实际工作流很关键：一旦实现，我们就能进一步构建“文生图 → 图生皮肤”的完整链路。因此，我们需要选择具备图生图能力的基础模型作为底模进行微调。

我们最终选择了 Flux2 Klein Base 4B 作为基础模型。
它是 Flux2 系列中较轻量的版本，在保持较好图像理解和泛化能力的同时，
对显存和计算资源的需求更加友好，适合做特定场景的 LoRA 微调，以快速验证想法的可行性。
未来可以使用参数量更大的模型继续提升效果（比如 Qwen Image Edit、Flux2 Klein Base 9B）。

## 四、关键问题：训练集设计与训练参数

训练集的设计核心在于：**在有限的分辨率下，通过集成高密度的多维信息，引导模型更稳定地捕捉从“写实人像”到“像素贴图”和渲染图的抽象映射。**

数据集由成对的 **控制图像（Control Image）** 与 **目标图像（Target Image）** 组成。

- **控制图像**：采用主体的前后写实照片。左侧为正面全身立像，右侧为背面全身像，为模型提供完整的特征条件参考。得益于 Flux2 Klein 相对较强的图像理解能力，在模型推理时，即使只传入正面照片，也有机会生成合理的皮肤背面。

![控制图像示例|800](/articles/images/8880005_control_img.jpg)

- **目标图像**：作为模型学习的最终目标。为了提升生成皮肤在 3D 空间中的一致性和细节表现，它集成了核心 UV 贴图与多视角的 3D 渲染图。

![目标图像示例|800](/articles/images/8880005.jpg)

目标图像的布局设计如下：

| 布局位置 | 详细内容 | 设计意图 |
| :--- | :--- | :--- |
| **左上角** | 完整的 UV 贴图（含透明像素标记） | 提供核心皮肤纹理数据，透明像素标记用于辅助恢复透明通道 |
| **右上角** | • 透视主视角的动态姿态预览<br>• 头部与躯干的高倍特写 | 细节强化：减少不对称瑕疵，优化复杂配饰（如卫衣、围巾）的生成质量 |
| **下半部分** | • 内/外层皮肤的全视角视图（正背、俯仰）<br>• 头部多角度细节特写 | 结构校验：帮助检查 3D 空间的一致性与视觉连贯性 |

在我们的实验中，布局位置本身没有表现出决定性影响；相比之下，信息密度、视角覆盖和 UV 贴图与 3D 渲染图之间的对应关系更关键。

虽然微调训练的过程带有一定的经验主义色彩，但在经历数十个版本的迭代后，我们总结出了一套当前较稳定的方法论：

1. 渐进式数据策略：首先尝试少量样本（20 张）进行训练，如果能学习到一些基本特征再增加样本。
   这样可以快速验证训练流程是否正确，避免在大数据集上浪费时间。

2. 训练集质量比数量更重要：
   准确的色彩、合理的人物特征抽象和数据多样性都很重要。我们使用精选的 50 对样本（50 张控制图像、50 张目标图像）训练出的模型，已经能生成一批可用样例。而更多的数据集需要谨慎添加，因为这可能会在提高一部分场景生成质量的同时，降低另一部分的生成质量。

3. 特定场景的针对性数据集：我们在迭代中发现，模型对某些特征的学习需要专门的强化。
   比如帽子、胡子、眼镜等装饰表现不佳时，就在样本中增加对应的数据。
   但过于复杂的皮肤需要删除，因为它们会干扰模型对常见特征的学习，除非使用更大的泛化能力更强的底模进行训练。

4. 渲染参数：
   
   影响学习效果的因素很多，其中渲染参数的影响尤其明显：Voxel 模式 / Plane 模式；透视 / 正交投影；只渲染内层 / 只渲染外层 / 内外层；人物姿态；各组件的旋转角度；光照开关；网格开关等。

   ![Plane 模式渲染示例|300](/articles/images/8880005_plane.jpg)
   ![Voxel 模式渲染示例|300](/articles/images/8880005_voxel.jpg)
   
   Plane 模式与 Voxel 模式的区别在外层皮肤的渲染方式，Plane 模式把外层皮肤渲染成一个悬浮的平面，而 Voxel 模式把外层皮肤的每个像素渲染成一个方块。
   
   ![正交投影渲染示例|300](/articles/images/8880005_ortho.jpg)
   ![透视投影渲染示例|300](/articles/images/8880005_perspective.jpg)
   
   正交投影对比透视投影，透视投影近大远小，正交投影大小不变。或许是这个原因，透视投影能提供深度信息作为补充，更容易让模型学习到皮肤在三维空间的映射关系。
   
   ![关闭光照的渲染示例|300](/articles/images/8880005_light_off.jpg)
   ![开启光照的渲染示例|300](/articles/images/8880005_perspective.jpg)
   
   光照开关对比，光照关闭时（左图），每个像素块显示原本的颜色，这使得相邻的同色像素块之间没有明显区别；光照开启后的阴影让渲染效果更立体，模型更容易学习到映射关系。虽然高光和反光会影响一些区域的色彩准确性（比如肩部），但在我们的观察中，开启光照后的结果更稳定。
   
   ![仅渲染内层皮肤|300](/articles/images/8880005_inner.jpg)
   ![仅渲染外层皮肤|300](/articles/images/8880005_overlay.jpg)
   ![同时渲染内外层皮肤|300](/articles/images/8880005_both.jpg)

   内外层的渲染方式、视角：内外层的渲染方式、视角也会影响模型对纹理映射的学习，应尽可能无死角地展示所有体素。（从左至右）仅渲染内层皮肤；仅渲染外层皮肤；同时渲染两层皮肤。我们需要结合每一种渲染方式，以帮助模型充分理解皮肤的结构和纹理。

### 训练收敛演变与生成表现

在 LoRA 微调过程中，我们记录了模型在不同训练步数（Steps）下的生成输出。从下方的演变过程可以清晰地观察到，模型是如何一步步从无序的噪声中学习到 Minecraft 皮肤复杂的 UV 布局与三维映射关系的：

#### 1. 早期阶段（500 - 2000 步）：初步布局学习与混沌阶段
在此期间，模型开始摸索输出的整体排版。前 1000 步的输出基本为模糊的色块与凌乱的线条，UV 贴图与渲染图的界限极度混乱。到 2000 步时，模型初步确立了“左上角为 UV 贴图、其余区域为 3D 渲染图”的复合布局规则，但细节结构依然处于混沌状态。

在此阶段，模型快速吸收了 UV 贴图的网格对齐特征。3D 角色的基本身体组件（头部、躯干、四肢）轮廓变得清晰，服装的主体色彩（如红蓝、黑白配色）也开始正确映射到对应组件上。尽管像素边缘依然模糊且有一定噪点，但整体骨架已经基本定型。

随着步数的增加，模型开始进入精细化特征的学习。五官轮廓、服饰小配件以及内外层皮肤（如发型层次、外套边缘）的立体结构被逐渐建立。用于后处理提取的透明像素标记（Alpha Marker）点阵开始呈现出规律的几何排布，生成的质量和一致性显著提升。

在 18000 步左右，图像的模糊感逐渐消除，像素点变得锐利，网格边界清晰。角色的面部神态、精细的服饰纹理（如衣领褶皱、鞋包）达到了比较高的拟合度。Alpha Marker 规律且清晰。

> 注：上面的演变过程基于一个早期的训练集设计，所以目标图像并不与本文中的设计一致 

## 五、高质量数据集的构建

构建可用的皮肤生成模型，关键在于高质量成对的 **控制图像-目标图像（Control-Target）** 数据集。目标图像通常相对容易获取：可以从社区获取 UV 贴图皮肤数据集，然后通过脚本生成目标图像。但匹配的写实风格控制图像则面临获取困难。为此，我们采用了以下两种策略：

#### 1. 逆向推导生成

利用多模态图像模型（如 GPT Image2）的视觉理解能力，从现有的皮肤贴图“反向推导”出对应的写实人像照片。

**推荐提示词示例：**

> “根据该 Minecraft 皮肤的特征，生成对应的真人全身正面站立照；并在图片右侧，生成其对应的背影照，尽量保持服饰与细节一致。”

#### 2. 自演化数据合成循环
构建一个自动化的生成管线，通过“模型辅助模型”的方式实现自演化迭代：

1. **文本特征描述**：利用 LLM 随机生成多样化的人物外貌与配饰描述。
2. **写实人像合成**：调用文生图模型生成对应的真人全身照。
3. **皮肤生成**：利用当前版本的“图生皮肤”模型生成对应的目标图像。
4. **筛选与修复**：对生成结果进行结构校验和评估（人工或自动），剔除或修复瑕疵数据。记录有瑕疵的场景，准备更多数据专门优化。
5. **数据集动态扩充**：将优化后的高质量数据对并入训练集，启动下一轮模型迭代。

这种闭环机制使我们能够快速找出薄弱场景并进行针对性的数据增强。

## 六、后处理：从目标图像中提取皮肤

由于扩散模型默认生成的图像为 RGB 格式，原生不包含透明通道，我们需要从生成结果中精确提取出 64×64 的像素皮肤贴图，并恢复外层皮肤的透明信息。
在训练时，如果将渲染图的背景统一设置为灰色（128, 128, 128），当生成的皮肤本身包含相近的灰色元素时，会导致部分前景像素被误判为背景。此外，生成图像在前景与背景的交界处往往会出现色彩的渐变融合，使得常规的纯色阈值提取法难以奏效。

![透明像素特征标记示例|400](/articles/images/8880005_alpha.jpg)

**解决方案：透明像素特征标记（Alpha Marker）**

我们设计了一种特殊的图像编码策略：在构建训练集时，于 UV 贴图的每个透明像素中心，预先绘制一个微小的白色矩形标记。模型经过学习后，会在生成结果中使用这些标记表达透明像素。
在后处理的提取阶段，可以通过较直接的算法检测这些白色的“特征锚点”：如果某个像素的中心区域接近白色，且边缘区域接近背景色，即可将其判定为透明像素。这种方法将棘手的“前景/背景分离”难题转化为更可控的“特征检测”问题。

不足之处在于，该方法仅支持分辨两种状态：全透明和不透明。而 RGBA 图像理论上可以表示 256 级透明度，适用于半透明眼镜等场景。如何让模型稳定生成多级透明度，仍然需要继续探索。

## 七、当前局限性与未来探索

当前方案已经能生成一批可用样例，但距离稳定产品化仍有距离，主要限制包括：

- **多级透明度尚未解决**：透明像素特征标记目前只能稳定表达“全透明 / 不透明”两种状态，尚不能可靠生成眼镜、薄纱、半透明发饰等多级透明材质。
- **复杂装饰生成的结果不稳定**：对于异形头饰服饰和跨组件装饰，可能生成不锐利的、断裂的、错位的或左右不对称的结果。
- **泛化能力仍待提升**：对于没有在训练集中出现过的动物，有可能产生瑕疵或动物特征丢失。
- **背面信息的推断结构不稳定**：当控制图像只提供正面视角时，模型需要推断背面服饰、发型和配件结构。这一推断可能与原始设定不一致。
- **后处理算法仍不完美**：Alpha Marker 的检测依赖颜色阈值和像素区域判断，仍可能出现误判。

基于这些局限性和当前的实践经验，我们计划在以下方向持续推进研究：

- **逆向生成模型**：从皮肤贴图反向推导出写实人像照。
- **控制图像的预处理增强**：在控制图像阶段引入预处理工作流，利用 AI 对参考图进行自动化姿态矫正（如强制直立）、背面特征预测以及前置的统一风格化处理，使模型更容易生成准确的皮肤。
- **LoRA 的基础上引入 DPO**： Direct Preference Optimization，直接偏好优化，增强模型生成的稳定性与质量
- **子艺术风格控制与边界探索**：持续摸索提示词工程与模型能力的边界。对于同样一张控制图像，可控地生成多种艺术风格的皮肤，例如 Low-Poly 低多边形风格、不同性别、不同种族的生成策略、体素边界过渡的可控（更锐利或者更平滑）。
- **探索更稳定的前景提取技术**：即使我们使用特殊的透明特征标记表示全透明和不透明，训练数据也只有这两种状态，模型仍然有概率生成纯色块与标记块的“中间状态”（半透明），导致最终皮肤的纹理出现背景噪声或者缺失。
- **完善自演化数据合成循环**：进一步完善自演化数据合成循环，使其能够自动生成更高质量、更多样化的数据集，从而持续提升模型性能。
- **基于更大参数的底模进行训练**：以期获得更高的生成质量和更强的泛化能力。

## 总结

Minecraft 皮肤生成看似只是把参考图转换成像素风格角色，但真正的难点在于：模型必须同时理解 2D UV 贴图、3D 方块角色结构、内外层透明关系，以及参考图中的身份特征。单纯依赖提示词很难稳定满足这些约束，因此我们选择通过图生图底模微调、复合目标图像设计和后处理算法，把问题拆成模型更容易学习和工程上更容易校验的几个部分。

从当前结果看，复合目标图像、少量高质量样本和透明像素特征标记是这套方案中最关键的三个环节。后续如果能进一步完善自动结构校验、失败样例回流和多级透明度处理，Minecraft 皮肤生成会更接近可持续迭代的工程系统，而不只是一次性的模型实验。

我们也欢迎对 Minecraft、生成模型、数据合成或像素艺术感兴趣的朋友一起推进新的模型和方法研究。如果你熟悉模型训练、推理部署或后处理算法，可以通过 GitHub、Hugging Face 或 Discord 参与我们的讨论与实验；如果你是非技术背景的爱好者，也可以直接体验我们的在线生成器。你所分享的参考图、生成结果和失败案例，都将帮助我们观察模型在哪些角色、风格和结构上仍然薄弱，并为后续的数据筛选、模型评测和训练提供极富价值的现实参考。
